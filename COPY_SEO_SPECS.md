## Brand Guidelines

### Voice Characteristics

- **Warm:** Friendly, approachable, like a helpful engineer neighbor
- **Technical:** Accurate terminology, respect for the craft
- **Slightly Irreverent:** Occasional humor, willingness to call out industry BS
- **Inclusive:** Speak to beginners and pros with equal respect

### Do's and Don'ts

| ✅ DO | ❌ DON'T |
|-------|----------|
| Use conversational language where appropriate | Be overly casual or unprofessional |
| Explain technical concepts simply | Dumb things down or talk down to users |
| Acknowledge the learning curve | Pretend everything is instant magic |
| Call out industry problems (expensive plugins, subscriptions) | Trash-talk competitors directly |
| Be honest about limitations | Make false promises |
| Use humor sparingly and deliberately | Try too hard to be funny |

### Tone by Section

| Section | Tone |
|---------|------|
| **Hero** | Confident, punchy, energizing |
| **Plugin Descriptions** | Technical but accessible, feature-focused |
| **CTA** | Clear, direct, low-friction |
| **Support/Docs** | Patient, educational, encouraging |
| **Pricing** | Transparent, no-nonsense, value-forward |
